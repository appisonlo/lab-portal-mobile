# lab-portal-mobile

## 项目简介

`lab-portal-mobile` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，旨在提供一个集中存放和展示各类静态页面的空间，方便快速访问与维护。

## 目录结构

```
lab-portal-mobile/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 共用资源文件（CSS、JavaScript、图片等）
├── index.html      # 入口页面，用于导航或列表展示
└── README.md       # 本文件
```

- **pages/**：每个 HTML 页面作为一个独立模块存放于此，文件名应具有描述性。
- **assets/**：存放页面间可复用的样式、脚本或媒体文件，保持资源集中管理。
- **index.html**：默认入口页面，可包含指向各页面的链接或简单说明。

## 页面归档说明

本仓库所收录的 HTML 页面均为独立功能模块，不依赖于外部特定域名或数据库。每个页面可单独打开运行，适用于记录实验性项目、工具页面或临时展示内容。

页面内容应保持简洁，不包含外部链接依赖（除非必要且稳定）。建议使用相对路径引用资源文件。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意页面。
3. 如需部署到静态服务器，将整个仓库目录上传至服务器根目录即可。

## 维护说明

- **添加新页面**：在 `pages/` 目录下创建新的 `.html` 文件，并更新 `index.html` 中的导航列表（如有）。
- **更新资源**：修改 `assets/` 下的文件时，请确保不影响已有页面的正常显示。
- **删除页面**：移除对应文件后，同步更新 `index.html` 中的引用。
- **版本记录**：建议在提交时简要说明新增或修改的页面内容，便于回溯。

## 注意事项

- 本仓库仅用于静态 HTML 页面的归档，不涉及后端处理或动态数据。
- 请勿在页面中嵌入跟踪脚本、广告或任何形式的第三方服务调用，保持纯粹性。
- 提交前请确保页面在不同设备或浏览器下基本可用（无需完美适配，但不应严重错乱）。

## 贡献

欢迎提交符合归档目的的 HTML 页面。请确保：
- 页面为独立文件，不依赖外部资源（或已内置资源）。
- 文件命名清晰，内容不违反相关法律法规。
- 提交 PR 时附上页面功能简介。
